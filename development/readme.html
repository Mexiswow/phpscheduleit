Welcome to phpScheduleIt 2.0!

This is a pre-release version of phpScheduleIt. It will certainly contain bugs and is not yet suitable for production use.

Prerequisites

  • PHP 5.2 or greater
  • MySQL 5.0 or greater

Server Configuration

Extract the contents of the .zip file to your web server. phpScheduleIt only requires that the /Web folder is accessible.
  • If possible, create a website or virtual directory pointed at the /Web folder.
  • If this is not possible, simply extract all of the contents to an accessible location on your server.
  • The PHP account requires read/write access to /tpl_c
  • To allow resource image uploads, the PHP account requires read/write access to your configured uploads directory ($conf['settings']['image.upload.directory'])

Application Configuration

The first step prior to running any installation process is to configure phpScheduleIt.
  • To do this first rename config/config.dist.php to config/config.php.
  • Then open config/config.php and alter any settings that do not match your environment.

Database Setup

Automated

The automated database setup only supports MySQL at this time. To run the automated database setup, navigate to the /Web/install directory in a web browser and follow the on screen instructions.

Manual

The packaged database scripts make assumptions about your desired database configuration. Please edit them to suit your environment before running.
Run the following scripts in order:
  • /database_schema/create-db.sql
  • /database_schema/schema-utf8.sql
  • /database_schema/create-user.sql
  • /database_schema/data-utf8.sql

The First Login

You will need to register an account with your configured admin email address. This is specified in the following config.php setting: $conf['settings']['admin.email']

Log Files

phpScheduleIt logs multiple levels of information categorized into either application or database logs. To do this, the log4php library is used. By default, logging is turned OFF.
  • First, rename /config/log4php.config.dist.xml to /config/log4php.config.xml
  • To allow application logging, the PHP account requires read/write access to your configured log directory
  • Logging is configured in /config/log4php.config.xml
  • Logging is controlled by changing the <level> of each <logger>
  • Levels used by phpScheduleIt are OFF, DEBUG, ERROR. For normal operation, ERROR is appropriate. If trace logs are needed, DEBUG is appropriate.
  • To turn on application logging, change the <level value="OFF" /> to an appropriate level for either the default or sql loggers. For example, <level value="ERROR" />
  • For more information on logging configuration, visit http://logging.apache.org/log4php/
