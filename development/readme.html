Welcome to phpScheduleIt 2.0! (A quick installation).

This is a pre-release version of phpScheduleIt. It will certainly contain bugs and is not yet suitable for production use.

Prerequisites

  • PHP 5.2 or greater.
  • MySQL 5.0 or greater.
  • Web server (e.g Apache2, etc...).
  • OS - Windows, Mac, Unix-like were tested.

(Note for users without web host service). If you want to accomplish all of the above quick and easy? try this apachefriends.org, it will do it for you in no time (recommended).

- http://www.apachefriends.org/en/index.html (highly recommended). Or
- http://netbeans.org/kb/docs/php/configure-php-environment-windows.html.

Server Deployment

  • Untar or unzip the distribution in your webserver's document root.
  • If you don't have direct access to your document root and use hosting service, put the files in a directory on your local machine, and transfer the directory on your web server using, for example, ftp.
  • Note: Some may have directory permission issues. Contact support below or contact your hosting service to resolve.

Application Configuration

You can configure phpScheduleIt to fit your environments and needs or use the minimal default settings.
  • You can use the minimal default pre-configuration. To allow resource image uploads, the web-server account requires read/write access to your configured uploads directory ($conf['settings']['image.upload.directory']), or you may leave it empty.
  • Or you can open config/config.php and alter any settings that do not match your environment.

Database Setup

You have 2 ways to set up your database for the application to work.

1) Automated

The automated database setup only supports MySQL at this time. To run the automated database setup, navigate to the /Web/install directory in a web browser and follow the on screen instructions.

OR

2) Manual

The packaged database scripts make assumptions about your desired database configuration. Please edit them to suit your environment before running. Run the following scripts in order (recommending phpmyadmin interface http://www.phpmyadmin.net/documentation/):
  • The web server must have write access to /your-phpScheduleIt/tpl_c and /your-phpScheduleIt/tpl (why?)
  • import "/database_schema/create-db.sql" to mysql db (phpscheduleit2 db created)
  • import "/database_schema/schema-utf8.sql" to phpscheduleit2 (db schema created)
  • import "/database_schema/create-user.sql" to mysql db (schedule_user/password mysql account created to match to your default configuration and notice: this is not the application user!). At this point with the aforementioned default configuration in your-phpScheduleIt/config/config.php your application can connect to the database and loaded eg. see http://localhost/your-phpScheduleIt/Web/index.php but you can not login since there is no application data or user, so go to next step
  • import "/database_schema/data-utf8.sql" to phpscheduleit2 (initializing application database)
  • import "/database_schema/testdata-utf8.sql" to phpscheduleit2 (a sample application data created with 2 users: 1) admin/password and 2) nkorbel/password). Use these users for testing your installation. You are done. Congratulation!

The First Login

  • Aforementioned if you imported a sample application data, you now can use 1) admin/password and 2) nkorbel/password
  • Or you will need to register an account with your configured admin email address. This is specified in the following config.php setting: $conf['settings']['admin.email']
    For the email registration process to work - this means it will send you and email with account login information. You need to make sure your email server is working and the application configured, look for smtp.* parameters, and fill them in properly. Examples are shown next the parameters

Log Files

phpScheduleIt logs multiple levels of information categorized into either application or database logs. To do this, the log4php library is used. By default, logging is turned OFF.
  • First, rename /config/log4php.config.dist.xml to /config/log4php.config.xml
  • To allow application logging, the PHP account requires read/write access to your configured log directory
  • Logging is configured in /config/log4php.config.xml
  • Logging is controlled by changing the <level> of each <logger>
  • Levels used by phpScheduleIt are OFF, DEBUG, ERROR. For normal operation, ERROR is appropriate. If trace logs are needed, DEBUG is appropriate.
  • To turn on application logging, change the <level value="OFF" /> to an appropriate level for either the default or sql loggers. For example, <level value="ERROR" />
  • For more information on logging configuration, visit http://logging.apache.org/log4php/
